Radio buttons

I've only seen this problem twice, but it did confuse me too back when I was still learning HTML.

Radio elements must share the same name if they relate to each other!

I've seen radio elements that all had the same name, and radio elements that didn't have any names.

Void and Non-Void tags

I see the confusion about these two way too often.

Some people still cannot draw the line between these two types of elements.

I'll either see a missing closing tag for a non-void element, or I'll see one for a void element.

I personally knew the difference between these two types of elements, but I actually hadn't known that they had their respective terms.

Anyways, always close your non-void tags, and optionally include a closing symbol ("/") in your void elements.

Non-void tags Void tags

General error checking

There are some students who definitely cannot recognize some errors within their code. And that's fine, for now at least.

Here is a list of things that students should review when cleaning up their code:

  1. Your non-void elements should have closing (and opening) tags.
  2. Your void elements should have no closing tags.
  3. Your code should follow proper indentation practices (below).
  4. You should use HTML entities (&entity;) instead of pasting symbols.
  5. Your commands should be spelled correctly.

These are some basic things to keep track of. If these are reviewed frequently, you can subconciously aquire better programming habits by making sure you aren't creating these problems in the first place.

Indentation

Indentation! I know it's not very important, but it shows how much you care about what you make! How much effort you're willing to put into your projects, how much you actually care about the user, how much you care about people reviewing your code — it's mostly psychological, and overall just good practice! If you're using a dedicated code editor (you are, it's called Visual Studio Code), then you barely need to worry about this. However, I still see people managing to fail at maintaining proper indentation! How? It's not even easy to get Visual Studio Code to accept inproper indentation!
